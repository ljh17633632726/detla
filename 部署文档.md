# 三角洲游戏平台 Docker 一键部署文档

> 文档更新：2026-07-17。当前部署包已包含 H5 微信支付、微信支付动态配置和余额充值套餐功能。

本目录是独立部署包，不依赖项目源码，包含：

- `delta-game-images.tar`：后端、管理后台、H5、MySQL、Redis 镜像（需单独下载）
- `docker-compose.yml`：容器编排文件
- `delta_game.sql`：数据库初始化脚本
- `start.bat` / `start.ps1`：Windows 一键启动
- `start.sh`：Linux 一键启动
- `stop.bat` / `stop.ps1` / `stop.sh`：停止服务
- `.env.example`：部署环境配置模板

## 最近更新

- H5 页面已接入微信 H5 支付，手机浏览器可直接跳转微信支付完成订单支付和余额充值。
- 增加 Docker 离线镜像包和 Windows/Linux 一键启动脚本，无需拉取项目源码即可部署运行。
- 微信支付配置已迁移到后台“系统配置”，修改 AppID、商户号、证书、回调地址等内容后动态生效，通常不需要重启后端。
- 增加余额充值套餐，后台可配置“充值金额、赠送金额、启用状态和排序”；用户从钱包进入充值页，支付成功后按“充值金额 + 赠送金额”入账。
- 余额充值支付单会保存当时的赠送金额，修改套餐不会影响已经发起的历史支付单。

## 一、运行条件

需要安装：

1. Docker Desktop（Windows）或 Docker Engine（Linux）
2. Docker Compose v2
3. 建议服务器至少 2 核 CPU、4GB 内存、40GB 可用磁盘
4. 生产环境需要一个公网域名和 HTTPS 证书

小程序不能通过 Docker 部署，需要使用微信开发者工具单独上传。

## 二、Windows 一键启动

1. 将整个部署目录复制到目标电脑。
2. 下载 `delta-game-images.tar` 并放到部署目录。
3. 打开 Docker Desktop，等待 Docker 引擎启动。
4. 将 `.env.example` 复制为 `.env`。
5. 修改 `.env` 中的数据库密码、Redis 密码、JWT 密钥和 AES 密钥。
6. 双击 `start.bat`。

启动后访问：

- 管理后台：`http://localhost:8081`
- H5 页面：`http://localhost:8082`

启动脚本会自动检查并加载同目录下的 `delta-game-images.tar`，然后执行 `docker compose up -d`。镜像包需要和 `docker-compose.yml`、启动脚本放在同一个部署目录。

## 三、Linux 一键启动

下载镜像并进入部署目录后执行：

```bash
cp .env.example .env
vi .env
chmod +x start.sh stop.sh
./start.sh
```

查看状态：

```bash
docker compose ps
```

查看后端日志：

```bash
docker compose logs -f backend
```

停止服务：

```bash
./stop.sh
```

停止服务不会删除数据库、Redis 和上传文件数据。

## 四、生产环境配置

### 1. 密码和密钥

必须修改 `.env` 中这些配置：

```env
MYSQL_ROOT_PASSWORD=修改为强密码
MYSQL_PASSWORD=修改为强密码
REDIS_PASSWORD=修改为强密码
JWT_SECRET=修改为随机长字符串
AES_KEY=修改为16字节AES密钥
```

### 2. 域名和 HTTPS

建议使用 Nginx、宝塔或云服务器反向代理：

```text
管理后台域名  -> 127.0.0.1:8081
H5域名        -> 127.0.0.1:8082
```

H5 和微信支付回调必须使用 HTTPS。微信支付回调地址为：

```text
https://你的域名/api/pay/wx/notify
```

### 3. 文件持久化

上传的商品图片、聊天图片、微信支付私钥和公钥会保存到 Docker 卷：

```text
upload-data
```

不要删除这个卷，否则会丢失上传文件和支付证书文件。

## 五、后台配置微信支付

启动后登录管理后台，打开“系统配置 -> 微信支付配置”，填写：

1. 启用微信支付
2. 微信支付 AppID
3. 商户号
4. 支付回调地址
5. API v3 密钥
6. 商户证书序列号
7. 微信支付公钥 ID
8. 上传商户私钥文件
9. 上传微信支付公钥文件
10. 点击保存配置

保存后支付配置动态生效，不需要重启容器。

H5 支付还需要填写 H5 客户端类型，普通手机浏览器使用 `WAP`。

### H5 微信支付额外要求

- H5 支付必须使用公网可访问的 HTTPS 域名，不能使用 `localhost` 或内网 IP。
- 微信支付回调地址必须填写完整地址，例如：`https://你的域名/api/pay/wx/notify`。
- H5 客户端类型普通手机浏览器填写 `WAP`；App 场景再按实际填写 `IOS` 或 `ANDROID`。
- 小程序支付的 AppID、H5 支付使用的 AppID 以及微信商户平台配置需要保持匹配。

### 微信支付动态配置说明

后台保存配置后，后端会动态读取最新配置并重建微信支付客户端，不需要为了修改支付证书或回调地址手动重启容器。商户私钥和微信支付公钥会保存到 `upload-data` Docker 卷，请勿删除该卷。

## 六、余额充值套餐

登录后台后进入“系统管理 -> 余额充值套餐”，可维护：

- 套餐名称
- 充值金额
- 赠送金额
- 排序和启用状态

默认套餐为 100 送 10、300 送 40、500 送 100。用户端进入“我的 -> 钱包 -> 立即充值”即可购买启用中的套餐。

充值成功后，系统会自动记录 `RECHARGE` 余额流水。已有数据库在升级到该版本后，如果 `payment` 表还没有 `recharge_bonus` 字段，请执行：

```sql
ALTER TABLE payment
  ADD COLUMN recharge_bonus decimal(10,2) NOT NULL DEFAULT 0.00 COMMENT '余额充值赠送金额' AFTER amount;
```

## 七、数据初始化说明

首次启动 MySQL 容器时会自动导入 `delta_game.sql`。

只有在 MySQL 数据卷为空时才会自动导入。后续修改 SQL 文件不会自动覆盖已有数据。

如果是全新测试环境，需要重新初始化数据，可以执行：

```bash
docker compose down -v
docker compose up -d
```

注意：`down -v` 会删除数据库、Redis 和上传文件数据，只能用于测试环境。

## 八、常用命令

```bash
# 启动
docker compose up -d

# 停止
docker compose down

# 查看容器
docker compose ps

# 查看全部日志
docker compose logs -f

# 重启后端
docker compose restart backend
```

## 九、安全提醒

- 不要把 `.env` 提交到代码仓库。
- 不要对公网开放 MySQL 3306 和 Redis 6379。
- 如果源码或旧配置中出现真实微信支付密钥，部署前应在微信商户平台更换。
- 生产环境必须配置 HTTPS、防火墙和定期备份。
