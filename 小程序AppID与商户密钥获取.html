小程序 AppID 与商户号密钥获取指南

本文档说明如何获取部署本系统所需的配置：小程序 AppID/AppSecret、商户号及其支付密钥、订单状态订阅消息模板。获取后填入 application-prod.yml 对应配置项即可。

一、小程序 AppID 与 AppSecret

1.1 登录微信公众平台

  1. 打开 微信公众平台
  2. 使用管理员微信扫码登录
  3. 进入你要部署的小程序管理后台

1.2 获取 AppID

  • 路径：开发管理 → 开发设置
  • 在「开发者 ID」区域可以看到 AppID，形如：wx1234567890abcdef

1.3 获取 AppSecret

  • 在同一页「开发者 ID」区域找到 AppSecret
  • 若未生成：点击「生成」
  • 若已泄露或忘记：点击「重置」，按提示完成管理员扫码等验证
  • 重要：生成/重置后会展示完整密钥，请立即复制保存。关闭页面后将无法再次查看，只能重新生成

1.4 配置示例

wx:
  miniapp:
    appid: wx1234567890abcdef    # 替换为你的 AppID
    secret: xxxxxxxxxxxxxxxxxxx  # 替换为你的 AppSecret

二、商户号

2.1 什么是商户号

商户号是微信支付分配给商家的唯一标识，开通微信支付后由微信提供，通常为一串纯数字。

2.2 获取商户号

  1. 打开 微信支付商户平台
  2. 使用商户号超级管理员账号登录
  3. 进入 账户中心 → 商户信息
  4. 在页面中可以看到 商户号（如 1234567890）
  5. 填入配置：wx.pay.mch-id

2.3 绑定小程序

在支付前，需将小程序 AppID 与商户号绑定：

  1. 商户平台 → 产品中心 → AppID 授权管理
  2. 点击「添加 AppID」
  3. 填入你的小程序 AppID，按提示完成授权

三、商户号密钥（APIv3 密钥）

3.1 设置 APIv3 密钥

  1. 商户平台 → 账户中心 → API 安全 → API 密钥
  2. 点击「设置 APIv3 密钥」
  3. 输入一个 32 位密钥（建议使用随机生成工具，可含大小写字母和数字）
  4. 完成验证后保存
  5. 重要：保存后无法再查看原文，只能重新设置。请务必妥善保存
  6. 填入配置：wx.pay.api-v3-key

四、商户 API 证书（私钥）

用于对支付请求进行签名，系统必需。

4.1 申请证书

  1. 商户平台 → 账户中心 → API 安全 → 申请 API 证书
  2. 按页面提示下载「证书生成工具」
  3. 运行工具，按提示操作（需输入商户号、商户名称等）
  4. 工具会生成以下文件：
    • apiclient_key.pem — 商户私钥，本系统必需
    • apiclient_cert.pem — 商户证书（本系统不使用）
    • 其他文件

4.2 获取证书序列号

  • 在证书工具完成后，或商户平台 API 安全 页面会展示 证书序列号
  • 形如：5F8A9B2C3D4E5F6A7B8C9D0E1F2A3B4C5D6E7F
  • 填入配置：wx.pay.cert-serial-no

4.3 部署私钥到服务器

  1. 将 apiclient_key.pem 上传到服务器
  2. 建议路径：/www/certs/apiclient_key.pem
  3. 填入配置：wx.pay.private-key-path

五、微信支付公钥（平台公钥）

用于验签微信的支付回调和响应。

5.1 申请公钥

  1. 商户平台 → 账户中心 → API 安全
  2. 找到「微信支付公钥」或「平台证书」相关入口
  3. 按提示申请/下载 微信支付公钥文件（通常为 pub_key.pem）

5.2 获取公钥 ID 与部署

  • 在「微信支付公钥」页面会展示 公钥 ID，形如：PUB_KEY_ID_01XXXXXXXXXXXXXXXX
  • 将公钥文件上传到服务器，建议路径：/www/certs/pub_key.pem
  • 填入配置：wx.pay.public-key-path、wx.pay.public-key-id

六、支付回调地址（notify-url）

  • 由你自行填写，必须是 HTTPS 且公网可访问
  • 格式示例：https://你的域名/api/pay/wx/notify
  • 填入配置：wx.pay.notify-url

七、订阅消息模板（订单状态变更）

用于向用户发送订单支付成功、被接单、已取消等重要状态通知，需在 微信公众平台 申请。

7.1 进入模板库

  1. 登录 微信公众平台
  2. 进入 功能 → 订阅消息
  3. 点击「公共模板库」

7.2 添加订单状态模板

  1. 在公共模板库中搜索「订单」「状态」或「通知」
  2. 找到合适的订单状态类模板，点击「选用」
  3. 选用时必须选择以下字段（字段名需与系统一致）：
    • phrase4：订单状态
    • character_string1：订单号
    • thing2：订单类型
    • amount3：金额
  4. 添加成功后，在「我的模板」列表中可看到该模板的 模板 ID
  5. 将模板 ID 填入 application-prod.yml 中 subscribe-template.order-status

7.3 配置示例

wx:
  miniapp:
    subscribe-template:
      order-status: __你的订单状态模板ID__

7.4 注意事项

  • 模板字段名（phrase4、character_string1、thing2、amount3）必须与系统要求一致
  • 用户需在小程序内主动订阅后，才能收到该模板消息

八、配置汇总表

配置项获取位置说明
appid公众平台 → 开发管理 → 开发设置小程序 AppID
secret同上AppSecret（生成后仅展示一次）
subscribe-template.order-status公众平台 → 功能 → 订阅消息订单状态变更模板 ID
mch-id商户平台 → 账户中心 → 商户信息商户号
api-v3-key商户平台 → API 安全 → API 密钥32 位密钥
private-key-path申请 API 证书后apiclient_key.pem 路径
cert-serial-no申请 API 证书后证书序列号
public-key-path微信支付公钥公钥文件路径
public-key-id同上公钥 ID
notify-url自行填写支付回调地址

九、完整配置示例

wx:
  miniapp:
    appid: wx1234567890abcdef
    secret: your_app_secret_here
    subscribe-template:
      order-status: __你的订单状态模板ID__

  pay:
    mch-id: 1234567890
    api-v3-key: your_32_char_api_v3_key_here
    private-key-path: /www/certs/apiclient_key.pem
    cert-serial-no: 5F8A9B2C3D4E5F6A7B8C9D0E1F2A3B4C5D6E7F
    public-key-path: /www/certs/pub_key.pem
    public-key-id: PUB_KEY_ID_01XXXXXXXXXXXXXXXX
    notify-url: https://你的域名.com/api/pay/wx/notify

十、注意事项

  1. AppSecret、APIv3 密钥 一旦生成/设置，页面关闭后将无法再次查看，请立即保存到安全位置
  2. 证书和密钥文件务必妥善保管，不要提交到公开代码仓库
  3. 商户号需先完成认证，并在「AppID 授权管理」中绑定小程序后方可正常支付
  4. order-status 订阅消息模板用于订单支付、接单、取消等重要状态推送；用户需在小程序内主动订阅后才能收到
