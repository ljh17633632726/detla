H5 与微信小程序切换说明

本文说明前端在 H5、微信小程序之间的构建切换，以及后端微信小程序、微信支付能力的启停配置。

1. 当前支持范围

能力 H5 微信小程序
用户登录 手机号 + 验证码登录 微信 uni.login + 首次手机号授权
客服登录 账号密码登录 账号密码登录
余额支付 支持 支持
微信支付 暂未接入 H5 在线支付 支持微信 JSAPI 支付
微信订阅消息 自动跳过 支持
后端启动是否依赖微信配置 不依赖 启用微信能力时依赖

2. 前端构建切换

前端项目目录：

cd delta-mp

H5 开发运行：

npm run dev:h5

H5 打包：

npm run build:h5

H5 构建产物：

delta-mp/unpackage/dist/build/h5

微信小程序开发构建：

npm run dev:mp-weixin

微信小程序打包：

npm run build:mp-weixin

微信小程序构建产物：

delta-mp/unpackage/dist/build/mp-weixin

用微信开发者工具导入上述 mp-weixin 目录即可运行小程序。

3. 后端开关配置

后端项目目录：

cd delta-game

后端需要 Java 17 或更高版本编译运行。当前机器如果默认是 Java 11，需要切换到 JDK 17+。

3.1 纯 H5 模式

纯 H5 模式不需要配置微信小程序 appid/secret，也不需要微信支付证书。

当前开发环境 H5 登录固定验证码为 123456，来源于 delta-game/delta-admin/src/main/resources/application-dev.yml 中的 auth.h5.login-code 配置。

wx:
  miniapp:
    enabled: false
  pay:
    enabled: false

auth:
  h5:
    login-code: 123456

效果：

  • 后端不会创建微信小程序 SDK。
  • /user/auth/login 微信登录接口会返回“微信小程序登录未启用，请使用H5手机号登录”。
  • 微信订阅消息自动跳过。
  • 后端不会创建微信支付 SDK，也不会读取支付证书文件。
  • /pay/wx/{orderId} 和 /pay/player-deposit 会返回微信支付未启用相关提示。
  • H5 用户使用 /user/auth/h5-login 登录。

3.2 微信小程序模式

如果要运行微信小程序登录和订阅消息，开启小程序配置：

wx:
  miniapp:
    enabled: true
    appid: wx_xxx
    secret: xxx
  pay:
    enabled: false

效果：

  • 小程序可以微信登录。
  • 首次登录仍要求授权手机号。
  • 可以发送微信订阅消息。
  • 微信支付仍关闭，不需要支付证书。

3.3 微信小程序 + 微信支付模式

如果要启用微信支付，必须同时配置小程序和支付参数：

wx:
  miniapp:
    enabled: true
    appid: wx_xxx
    secret: xxx
  pay:
    enabled: true
    mch-id: xxx
    notify-url: https://your-domain.com/api/pay/wx/notify
    api-v3-key: xxx
    private-key-path: /path/to/apiclient_key.pem
    cert-serial-no: xxx
    public-key-path: /path/to/pub_key.pem
    public-key-id: xxx

效果：

  • 后端启动时会加载微信支付 SDK。
  • private-key-path 和 public-key-path 指向的文件必须真实存在。
  • 小程序端可以调用微信支付。
  • 支付回调地址必须能被微信支付平台访问。

4. 生产环境建议

生产环境不要把开发环境固定验证码 123456 写死在配置文件中。推荐用环境变量：

auth:
  h5:
    login-code: ${H5_LOGIN_CODE:}

如果 H5_LOGIN_CODE 为空，H5 登录接口会返回“H5登录验证码未配置”。正式上线建议接短信验证码服务，而不是使用固定验证码。

微信支付生产环境建议：

  • wx.pay.enabled=true 只在确实要使用微信支付的环境开启。
  • 支付证书文件放在服务器固定目录，例如 /www/certs/。
  • 证书目录权限只开放给后端运行用户读取。
  • 不要把真实商户密钥提交到代码仓库。

5. 登录账号合并规则

H5 用户使用手机号注册后，用户再从微信小程序首次登录并授权同一手机号时，后端会把微信 openid 绑定到原 H5 用户，避免生成两个账号。

规则：

  • H5 首次登录时，openid 会临时保存为 h5:{phone}。
  • 小程序首次登录授权手机号后，如果找到同手机号 H5 用户，会把该用户的 openid 更新为真实微信 openid。
  • 如果该手机号已经绑定其他真实微信 openid，会拒绝绑定并提示“该手机号已绑定其他微信账号”。

6. 常见问题

不配置微信支付证书，Java 后台能启动吗？

可以。设置：

wx:
  pay:
    enabled: false

关闭后不会读取 apiclient_key.pem 或 pub_key.pem。

不配置微信 appid/secret，Java 后台能启动吗？

可以。设置：

wx:
  miniapp:
    enabled: false

关闭后微信登录不可用，但 H5 手机号登录可用。

H5 为什么不能直接用小程序微信支付？

小程序微信支付使用 JSAPI 支付参数和小程序 openid。H5 在线支付需要单独接微信 H5 支付或公众号支付链路，不能直接复用当前小程序支付接口。

H5 运行时请求哪个后端地址？

前端接口基地址来自：

VITE_API_BASE_URL

如果没有配置，默认请求：

http://localhost:8080

本地开发可以在 delta-mp/.env.development 中配置。
